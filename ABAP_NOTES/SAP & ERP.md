# SAP & SAP ABAP – Quick Interview Notes

## What is SAP?
- **SAP** = Systems, Applications, and Products in Data Processing.
- Founded in **1972** by **five former IBM engineers**.
- SAP SE is a German multinational software company.
- SAP is one of the world's leading **ERP software** providers.
- SAP integrates all business processes using a **centralized database**.

---

## What is ERP?
- **ERP** = Enterprise Resource Planning.
- ERP integrates all departments into one system.
- ERP provides **real-time information**.
- ERP eliminates duplicate data.
- ERP improves productivity and decision-making.

**Interview Definition:**
> ERP is a business management system that integrates all departments of an organization using a centralized database.

---

## ERP Full Form
- **E** – Enterprise (Organization)
- **R** – Resources (Money, Manpower, Materials, Machines)
- **P** – Planning (Efficient utilization of resources)

---

## What is ABAP?
- **ABAP** = Advanced Business Application Programming.
- SAP's programming language.
- Used to develop and customize SAP applications.
- Used to implement business logic.

---

## Responsibilities of an ABAP Developer
- Develop Reports.
- Create Interfaces.
- Perform Data Conversions.
- Build Enhancements.
- Design Forms.
- Develop Workflows.
- Create Database Tables.
- Write Open SQL Queries.
- Debug Programs.

---

---

## SAP GUI
- GUI = Graphical User Interface.
- Used to access the SAP system.
- Used to execute Transaction Codes (T-Codes).
- Used to create and execute reports.

---

## SAP HANA
- HANA = High-Performance Analytic Appliance.
- SAP's in-memory database.
- Provides high-speed data processing.
- Supports real-time analytics.

---

## SAP S/4HANA
- Latest SAP ERP suite.
- Runs on SAP HANA.
- Faster than SAP ECC.
- Simplified data model.
- Supports SAP Fiori.

---

## Master Data
- Data that rarely changes.
- Examples:
  - Customer
  - Vendor
  - Material
  - Employee

---

## Transaction Data
- Data created during daily business activities.
- Examples:
  - Sales Order
  - Purchase Order
  - Invoice
  - Payment
